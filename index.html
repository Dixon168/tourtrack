The Problem

Tour companies are still running
on WhatsApp and spreadsheets.

When agents call in bookings, mistakes happen. Overbooking, lost records, no visibility — it costs money and reputation.

😩
Without TourTrack
  • Agents book by WhatsApp — easy to miss or double-book
  • No real-time seat availability across agents
  • Passenger records scattered in notebooks and chats
  • No way to track which agent sold what
  • Month-end reconciliation is a nightmare
  • Managers have zero visibility until it's too late
🚀
With TourTrack
  • Agents book online — seats update instantly for everyone
  • Live seat count visible to all agents at all times
  • Full passenger roster with rooms, passports, ages, gender
  • Every booking tagged to the agent who made it
  • Finance reports by agent, by tour, by date — one click
  • Admin has full control and visibility, always

Features

Everything you need.
Nothing you don't.

📅
Batch Tour Scheduling
Create a whole season of tours in seconds. Pick a date range, select which days of the week, preview and confirm — done.
👥
Passenger Management
Full roster per booking — name, age, gender, passport, room number. Organized by family group and room, ready for the bus or hotel.
💰
Agent Finance Tracking
See exactly who sold what. Lookup by date range or by tour. Every booking tagged to the agent — perfect for commission reconciliation.
🔴
Real-time Availability
Seat counts update the moment a booking is made. Every agent sees the same live number — no more overbooking.
📊
Booking Lookup
Search bookings by date range or by tour. Export to CSV. Agents see only their own, admins see everything.
🏢
Multi-Merchant Platform
Run multiple tour companies from one platform. Each merchant has their own data, agents, and admin — fully isolated.

One-time setup — takes about 15 minutes. Free forever.
1Create a free Supabase account
Go to supabase.com → click Start your project → sign up with GitHub or email → create a new project (any name, any region).
2Run this SQL to create your tables
In your Supabase project → click SQL Editor in the left sidebar → paste the code below → click Run.
-- Run this entire block in Supabase SQL Editor create table if not exists tt_users ( id bigserial primary key, name text not null, username text unique not null, code text not null, role text not null default 'user', created_at timestamptz default now() ); create table if not exists tt_categories ( id bigserial primary key, name text not null, color text not null default '#58a6ff', created_at timestamptz default now() ); create table if not exists tt_tours ( id bigserial primary key, name text not null, date text not null, time text not null, duration text not null, type text not null default 'Bus', cat_id bigint references tt_categories(id), total integer not null, sold integer not null default 0, created_at timestamptz default now() ); create table if not exists tt_history ( id bigserial primary key, tour_id bigint references tt_tours(id), tour_name text not null, qty integer not null, av_after integer not null, booked_by text not null, user_id bigint references tt_users(id), agent text default '', ref text default '', notes text default '', created_at timestamptz default now() ); -- Disable RLS so the anon key can read/write alter table tt_users enable row level security; alter table tt_categories enable row level security; alter table tt_tours enable row level security; alter table tt_history enable row level security; create policy "allow all" on tt_users for all using (true) with check (true); create policy "allow all" on tt_categories for all using (true) with check (true); create policy "allow all" on tt_tours for all using (true) with check (true); create policy "allow all" on tt_history for all using (true) with check (true); -- Insert default admin account insert into tt_users (name, username, code, role) values ('Admin', 'happy168', '168168', 'admin') on conflict (username) do nothing; -- Insert sample category and tour insert into tt_categories (name, color) values ('New England', '#58a6ff') on conflict do nothing; insert into tt_tours (name, date, time, duration, type, cat_id, total, sold) select 'Boston 2 Day 1 Night', '2026-04-24', '08:00', '2 Days', 'Bus', id, 55, 40 from tt_categories where name = 'New England' limit 1;
